## Learned User Preferences

- Prefer concise, complete package READMEs (install, usage, tools, scripts, migration notes when relevant); omit LangChain in published READMEs—keep upstream prompt attribution in source only.
- `@eyueldk/aisdk-toolkit-browser` uses **Playwright** internally (not a consumer dependency). **`createBrowserToolkit({ browserWSEndpoint? })`** returns **`{ tools, hint, state: { browser: BrowserInstance } }`**; tools take optional **`pageId`** (string id); omit for default page. Runtime uses **`BrowserInstance`** / **`ContextInstance`** / **`PageInstance`**; **`PageInstance`** attaches console/network ring-buffer inspectors on construction (no start/stop). Truncate large inspect fields only in **`inspectConsole`** / **`inspectNetwork`** tool output, not in inspectors. **`await state.browser.close()`** on teardown. **`format: "markdown"`** uses **`turndown`** + **`@truto/turndown-plugin-gfm`** (`htmlToMarkdown`).
- In GitHub Actions, do not add a separate global npm upgrade for OIDC trusted publishing; rely on the npm shipped with the selected Node version.
- Prefer [pnpm/action-setup](https://github.com/pnpm/action-setup) for installs after `actions/setup-node`, using `run_install` (e.g. frozen lockfile) and caching when useful.
- Publish automation: reusable workflow parameterized by workspace package name; concrete workflows for `@eyueldk/aisdk-toolkit-browser`, `@eyueldk/aisdk-toolkit-todos`, `@eyueldk/aisdk-toolkit-filesystem`, `@eyueldk/aisdk-toolkit-shell`, and `@eyueldk/aisdk-toolkit-fetch` on pushes to `main` with package-scoped path filters (e.g. `packages/browser/**`), not repo-wide lockfile/workspace files unless explicitly requested.
- Keep the root `.cursor/hooks` tree out of version control (gitignore).
- Use one MIT `LICENSE` at the repo root only; omit per-package `LICENSE` files from published workspace packages and point README license links at the root file on GitHub.
- OpenRouter agent integration tests live in each package as `tests/integration.test.ts` using AI SDK `ToolLoopAgent` and `@openrouter/ai-sdk-provider` (devDependency per package); helpers stay inlined per file (no shared repo `test/` folder). They run only when `OPENROUTER_API_KEY` and `OPENROUTER_MODEL` are non-empty (root `.env` via `dotenv` in each package `vitest.config.ts`; see `.env.example`). CI skips them without secrets. Browser integration still needs Playwright Chromium when enabled.
- Root `.env.example` documents integration-test env vars (`OPENROUTER_*`, `DAYTONA_API_KEY`, `DAYTONA_API_URL`); keep real values in gitignored `.env` only.
- Co-locate toolkit/tool option types with their factories (e.g. `tools/index.ts`); avoid separate `*-options.ts` files.
- **`@eyueldk/aisdk-toolkit-filesystem`** **`FileSystemAdapter`** uses Node **`stream.Readable`** / **`stream.Writable`** from **`createReadStream`** / **`createWriteStream`**; **`readFile`** / **`writeFile`** default to **`Buffer`**; pass **`{ encoding: "utf8" }`** for string I/O. AI SDK tools use utf8 encoding on the adapter.

## Learned Workspace Facts

- Private monorepo root package is `@eyueldk/aisdk-toolkit`; published workspace packages are `@eyueldk/aisdk-toolkit-browser` (`packages/browser`), `@eyueldk/aisdk-toolkit-todos` (`packages/todos`), `@eyueldk/aisdk-toolkit-filesystem` (`packages/filesystem`), `@eyueldk/aisdk-toolkit-shell` (`packages/shell`), and `@eyueldk/aisdk-toolkit-fetch` (`packages/fetch`).
- Package and docs use GitHub `https://github.com/eyueldk/aisdk-toolkit` for repository, issues, and homepage links.
- `.github/workflows/publish.browser.yml`, `publish.todos.yml`, `publish.filesystem.yml`, `publish.shell.yml`, and `publish.fetch.yml` run on `main` when the matching `packages/*/**` tree changes (and via `workflow_dispatch`); each calls `.github/workflows/reusable.publish.yml` (Node 24, `pnpm/action-setup@v6` frozen lockfile, `check`, `build`, `test`, `publish --no-git-checks`). Each package `prepublishOnly` runs `pnpm check && pnpm build`.
- Root `pnpm test` runs `@eyueldk/aisdk-toolkit-browser` then `@eyueldk/aisdk-toolkit-todos` then `@eyueldk/aisdk-toolkit-filesystem` then `@eyueldk/aisdk-toolkit-shell` then `@eyueldk/aisdk-toolkit-fetch` in sequence so parallel Vitest plus Playwright does not time out.
- `createTodosToolkit` returns `{ tools, hint, state }` where `state` is a serializable `TodoState` (`{ todos }`). `createFileSystemToolkit` / `createFileSystemTools` take `{ adapter, permissions? }`; toolkit adds `hint` and returns `{ adapter, permissions }` on `state`. Merge `tools` and append `hint` to system/instructions.
- `@eyueldk/aisdk-toolkit-filesystem` ships **`MemoryFileSystem`**, **`LocalFileSystem`**, **`DockerFileSystem`**, and **`DaytonaFileSystem`** (each **`await …create(...)`** — memfs, host `root`, dockerode container `root`, or Daytona sandbox with default **`root`** `workspace`). Adapter paths use **`resolvePath`**; **`..` is allowed when the resolved path stays inside `root`**; **`LocalFileSystem`** uses **`realpath`** so symlinks cannot escape **`root`**. **`DockerFileSystem`** **`readDir`** / **`readDirRecursive`** list via in-container **`find`** (GNU `-printf` or BusyBox `-type f`/`-type d`); read/write use **`getArchive`** / **`putArchive`**. **`DaytonaFileSystem`** read/write/list via **`@daytonaio/sdk`** (**`DAYTONA_API_KEY`**, optional **`DAYTONA_API_URL`** for self-hosted). Self-hosted OSS FS ops need a reachable sandbox toolbox proxy (**`proxy.localhost`** — map to loopback if DNS fails). Docker adapter tests use **Testcontainers** (`alpine`) when Docker is available; Daytona tests skip unless the key is set and a **`listFiles('workspace')`** toolbox probe succeeds.
- `@eyueldk/aisdk-toolkit-filesystem` permission rules are **first-match** (`enforcePermissions` / `evaluatePermission`). **`glob`** / **`grep`** use **`collectReadableFilePaths`**; **`list`** uses **`collectVisibleEntries`** so denied subtrees are not traversed.
- `@eyueldk/aisdk-toolkit-todos` uses camelCase AI SDK tool names (e.g. `writeTodos`, `readTodos`); todo status literals are `pending`, `in_progress`, and `completed`.
- `@eyueldk/aisdk-toolkit-filesystem` **`FileSystemAdapter.readDir(path)`** returns **`FileStat[]`** (immediate children); **`readDirRecursive(path)`** buffers a flat subtree; **`ls(path, { recursive?, stream? })`** wraps those or yields **`AsyncIterable<FileStat>`** when **`stream: true`**. Default **`glob`** / **`grep`** walk with **`ls(".", { recursive: true, stream: true })`**.
- `@eyueldk/aisdk-toolkit-browser` exports **`createBrowserToolkit`**, **`BrowserInstance`**, and related toolkit types; no public Playwright types.
- `@eyueldk/aisdk-toolkit-shell` ships **`LocalShell`**, **`DockerShell`**, **`SshShell`**, and **`DaytonaShell`** (each **`await …create(...)`**). **`ShellAdapter.exec(command, { cwd?, env?, timeoutMs?, stdin?, stdout?, stderr? })`** returns **`{ stdout, stderr, exitCode, signal }`** (default timeout **`DEFAULT_SHELL_TIMEOUT_MS`** = 120_000). Pass **`stdin`** as a string or **`Readable`**; pass **`stdout`** / **`stderr`** **`Writable`** streams to stream output (result strings are empty for streamed stdio). **`stdin`** string/stream on local/SSH only; not on Docker/Daytona. **`DaytonaShell`** uses **`sandbox.process.executeCommand`** (default **`cwd`** `workspace`; **`DAYTONA_API_KEY`**, optional **`DAYTONA_API_URL`**). **`stderr`** is always empty (combined stdout from the Daytona API). **`SshShell.dispose()`** closes the SSH session. Tool: **`runCommand`** via **`createShellToolkit({ adapter })`**. Docker/SSH adapter tests use **Testcontainers** when Docker is available; Daytona tests skip unless the key is set and a sandbox command probe succeeds.
- `@eyueldk/aisdk-toolkit-fetch` **`createFetchToolkit({ fetch?, defaultTimeoutMs? })`** uses **`globalThis.fetch`** by default; **`state`** holds **`{ fetch, defaultTimeoutMs }`**. **`fetchRequest`** takes **`path`** + optional **`query`** (`buildRequestUrl` via **`URL`**); **`performHttpFetch`** returns **`{ status, statusText, headers, body }`** (default timeout **`DEFAULT_FETCH_TIMEOUT_MS`** = 30_000). Optional **`format`**: `raw` (default) or `markdown` (HTML bodies via **`turndown`** + **`@truto/turndown-plugin-gfm`**).
