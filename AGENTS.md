## Learned User Preferences

- Prefer concise, complete package READMEs (install, usage, tools, scripts, migration notes when relevant); omit LangChain in published READMEs—keep upstream prompt attribution in source only.
- `@eyueldk/aisdk-toolkit-browser` uses **Playwright** internally (not a consumer dependency). **`createBrowserToolkit({ browserWSEndpoint? })`** returns **`{ tools, hint, browser: BrowserInstance }`**; tools take optional **`pageId`** (string id); omit for default page. Runtime uses **`BrowserInstance`** / **`ContextInstance`** / **`PageInstance`**; **`PageInstance`** attaches console/network ring-buffer inspectors on construction (no start/stop). Truncate large inspect fields only in **`inspectConsole`** / **`inspectNetwork`** tool output, not in inspectors. **`await browser.close()`** on teardown.
- In GitHub Actions, do not add a separate global npm upgrade for OIDC trusted publishing; rely on the npm shipped with the selected Node version.
- Prefer [pnpm/action-setup](https://github.com/pnpm/action-setup) for installs after `actions/setup-node`, using `run_install` (e.g. frozen lockfile) and caching when useful.
- Publish automation: reusable workflow parameterized by workspace package name; concrete workflows for `@eyueldk/aisdk-toolkit-browser`, `@eyueldk/aisdk-toolkit-todos`, and `@eyueldk/aisdk-toolkit-filesystem` on pushes to `main` with package-scoped path filters (e.g. `packages/browser/**`), not repo-wide lockfile/workspace files unless explicitly requested.
- Keep the root `.cursor/hooks` tree out of version control (gitignore).
- Use one MIT `LICENSE` at the repo root only; omit per-package `LICENSE` files from published workspace packages and point README license links at the root file on GitHub.
- OpenRouter agent integration tests live in each package as `tests/integration.test.ts` using AI SDK `ToolLoopAgent` and `@openrouter/ai-sdk-provider` (devDependency per package); helpers stay inlined per file (no shared repo `test/` folder). They run only when `OPENROUTER_API_KEY` and `OPENROUTER_MODEL` are non-empty (root `.env` via `dotenv` in each package `vitest.config.ts`; see `.env.example`). CI skips them without secrets. Browser integration still needs Playwright Chromium when enabled.
- Co-locate toolkit/tool option types with their factories (e.g. `tools/index.ts`); avoid separate `*-options.ts` files.
- **`@eyueldk/aisdk-toolkit-filesystem`** **`FileSystemAdapter`** uses Node **`stream.Readable`** / **`stream.Writable`** from **`createReadStream`** / **`createWriteStream`**; **`readFile`** / **`writeFile`** default to **`Buffer`**; pass **`{ encoding: "utf8" }`** for string I/O. AI SDK tools use utf8 encoding on the adapter.

## Learned Workspace Facts

- Private monorepo root package is `@eyueldk/aisdk-toolkit`; published workspace packages are `@eyueldk/aisdk-toolkit-browser` (`packages/browser`), `@eyueldk/aisdk-toolkit-todos` (`packages/todos`), and `@eyueldk/aisdk-toolkit-filesystem` (`packages/filesystem`).
- Package and docs use GitHub `https://github.com/eyueldk/aimachine` for repository, issues, and homepage links.
- `.github/workflows/publish.browser.yml`, `publish.todos.yml`, and `publish.filesystem.yml` run on `main` when the matching `packages/*/**` tree changes (and via `workflow_dispatch`); each calls `.github/workflows/reusable.publish.yml` (Node 24, `pnpm/action-setup@v6` frozen lockfile, `check`, `build`, `test`, `publish --no-git-checks`). Each package `prepublishOnly` runs `pnpm check && pnpm build`.
- Root `pnpm test` runs `@eyueldk/aisdk-toolkit-browser` then `@eyueldk/aisdk-toolkit-todos` then `@eyueldk/aisdk-toolkit-filesystem` in sequence so parallel Vitest plus Playwright does not time out.
- `createTodosToolkit` returns `{ tools, hint, state }` where `state` is a serializable `TodoState` (`{ todos }`). `createFileSystemToolkit` / `createFileSystemTools` take `{ adapter, permissions? }`; toolkit adds `hint` and returns `{ adapter, permissions }` on `state`. Merge `tools` and append `hint` to system/instructions.
- `@eyueldk/aisdk-toolkit-filesystem` ships **`MemoryFileSystem`**, **`LocalFileSystem`**, and **`DockerFileSystem`** (each **`await …create(...)`** — memfs, host `root`, or dockerode container `root`). Adapter paths use **`resolvePath`**; **`..` is allowed when the resolved path stays inside `root`**; **`LocalFileSystem`** uses **`realpath`** so symlinks cannot escape **`root`**. **`DockerFileSystem`** **`readDir`** / **`readDirRecursive`** list via in-container **`find`** (GNU `-printf` or BusyBox `-type f`/`-type d`); read/write use **`getArchive`** / **`putArchive`**. Docker adapter tests use **Testcontainers** (`alpine`) when Docker is available.
- `@eyueldk/aisdk-toolkit-filesystem` permission rules are **first-match** (`enforcePermissions` / `evaluatePermission`). **`glob`** / **`grep`** use **`collectReadableFilePaths`**; **`list`** uses **`collectVisibleEntries`** so denied subtrees are not traversed.
- `@eyueldk/aisdk-toolkit-todos` uses camelCase AI SDK tool names (e.g. `writeTodos`, `readTodos`); todo status literals are `pending`, `in_progress`, and `completed`.
- `@eyueldk/aisdk-toolkit-filesystem` **`FileSystemAdapter.readDir(path)`** returns **`FileStat[]`** (immediate children); **`readDirRecursive(path)`** lists a flat subtree (DFS via **`readDir`**); **`ls(path, { recursive? })`** wraps those. Default **`glob`** / **`grep`** use **`ls(".", { recursive: true })`** and filter **`type === "file"`**.
- `@eyueldk/aisdk-toolkit-browser` exports **`createBrowserToolkit`**, **`BrowserInstance`**, and related toolkit types; no public Playwright types.
